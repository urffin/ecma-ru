4 Общие сведения

Этот раздел содержит ненормативный обзор языка ECMAScript.

ECMAScript – это объектно-ориентированный язык программирования для выполнения вычислений и манипулирования вычислительными объектами в среде хоста. ECMAScript, в том виде, как он определен здесь, не является самодостаточным с вычислительной точки зрения; действительно, в этой спецификации нет положений для ввода внешних данных или вывода вычисленных результатов. Вместо этого ожидается, что вычислительная среда программы ECMAScript предоставит не только объекты и другие средства, описанные в данной спецификации, но также определенные объекты, специфичные для среды, описание и поведение которых выходят за рамки данной спецификации, за исключением указания, что они могут предоставлять определенные свойства, к которым можно получить доступ, и определенные функции, которые можно вызывать из программ на ECMAScript.

ECMAScript изначально был разработан для использования в качестве языка сценариев, но стал широко использоваться в качестве универсального языка программирования. Язык сценариев – это язык программирования, который используется для манипулирования, настроики и автоматизации использования возможностей существующей системы. В таких системах полезный функционал уже доступен через пользовательский интерфейс, и язык сценариев представляет собой механизм позволяющий управлять этой функциональностью из программы. Таким образом, существующая система, как говорят, обеспечивает хост-среду объектов и средств, которые дополняют возможности языка сценариев. Язык сценариев предназначен для использования как профессиональными, так и непрофессиональными программистами.

ECMAScript изначально был разработан как язык веб-сценариев, предоставляющий механизм для оживления веб-страниц в браузерах и запускать выполнение вычислений на сервера как части клиент-серверной архитектуры основанной на Web. Cейчас ECMAScript используется для обеспечения основных возможностей сценариев для различных хост-сред. Поэтому основной язык указан в этом документе отдельно от любой конкретной хост-среды.

Использование ECMAScript вышло за рамки простого написания сценариев и теперь используется для полного спектра задач программирования в самых разных окружениях. По мере расширения использования ECMAScript расширяются и возможности, которые он предоставляет. ECMAScript теперь является полнофункциональным языком программирования общего назначения.

Некоторые возможности ECMAScript похожи на те, которые есть в других языках; например C, Java™, Self, и Scheme, которые описаны в следующих документах:

  • ISO/IEC 9899:1996, Programming Languages – C.
  • Gosling, James, Bill Joy and Guy Steele. The Java™ Language Specification. Addison Wesley Publishing Co., 1996.
  • Ungar, David, and Smith, Randall B. Self: The Power of Simplicity. OOPSLA '87 Conference Proceedings, pp. 227-241, Orlando, FL, October 1987.
  • IEEE Standard for the Scheme Programming Language. IEEE Std 1178-1990.

4.1 Web сценарии

Веб браузеры предоставляют хост-среду для вычислений на стороне клиента, включающие в себя, например, объекты представляюшие окна, меню, высплывающие окна, диалоги, текстовые области, якоря, фреймы, историю, cookies, и ввод/вывод. Кроме того, хост-среда позволяет добавлять код скрипта к таким событиям как изменение фокуса, загрузка страницы и изображений, выгрузка документа, ошибки и прерывание загрузки ресурсов, выделение текста, отправка формы, и действия мыши. Код сценария заключается в рамки HTML и отображаемой страницы, представляющей собой комбинацию элементов пользовательского интерфейса, фиксированного и вычисляемого текста, и изображений. Код сценария реагирует на действия пользователя, что не требует наличия точки входа в программу.

Веб-сервер предоставляет другую хост-среду для вычислений на стороне сервера, включающие в себя, например, объекты, представляющие запросы, клиентов и файлы; и механизмы для блокировки и обмена данными. Совместное использование сценариев на стороне браузера и на стороне сервера позволяет распределять вычисления между клиентом и сервером, предоставляя настраиваемый пользовательский интерфейс для веб-приложения.

Каждый веб-браузер и сервер, которые поддерживают ECMAScript, предоставляют свою собственную хост-среду, дополняя среду выполнения ECMAScript.
